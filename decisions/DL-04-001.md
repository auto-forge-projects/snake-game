# DL-04-001: Snake çözüm yığını — Canvas+rAF render, minimal Express static, ince localStorage wrapper

- Tarih: 2026-07-19
- Faz: 4 — Çözüm Analizi
- Mod: AUTOPILOT
- Karar: İstemci-taraflı statik Snake için üç mimari seçim birleşik yığın olarak kararlaştırıldı: (1) render/oyun döngüsü HTML5 Canvas + `requestAnimationFrame`; (2) servis katmanı minimal Express + `express.static` + `/health` (coinflip/dice-game kalıbı); (3) skor kalıcılığı ince `localStorage` wrapper modülü.
- Değerlendirilen alternatifler: (1) DOM/CSS-grid hücreleri, ayrıca SVG; (2) çıplak Node `http`+`fs`; (3) doğrudan `localStorage` API çağrısı.
- Gerekçe: (1) Canvas tek-yüzey redraw ile NFR-2 (~60 FPS) ve NFR-1 (≤100ms) için takılmasız, deterministik performans verir; DOM/SVG çok-düğüm repaint'te FPS riski taşır. (2) `express.static` path-traversal'a karşı battle-tested — elle `fs` servisinin klasik açığını önler (NFR-3); iki mod da NFR-7 ≤150MB'ı karşılar; kalıp tutarlılığı deploy riskini düşürür. (3) Wrapper, Safari özel-mod `setItem` throw / quota / devre-dışı depolamada NFR-6 zarif düşüşü ve integer doğrulaması sağlar (~15 satır ek maliyet).
- Riskler: Canvas erişilebilirlik doğal DOM'a göre ek `aria-live` katmanı gerektirir (NFR-5) → Faz 6/9'da skor & oyun-sonu için ayrı `aria-live` bölgesiyle izlenecek. Express bağımlılığı `npm audit` yüzeyi ekler → CI'da Critical/High=0 kapısıyla (NFR-3) izlenir.
- Geri alınabilirlik: Yüksek — render katmanı oyun mantığından izole (Canvas↔DOM değişimi lokal); servis tek dosya (Express↔http değiştirilebilir); wrapper tek modül. Hiçbiri veri şeması/dış API kilidi yaratmaz.
